README 0.7.2

OpenSLP 0.7.2 contains several important bug fixes, framework changes (that will continued through 1.0 release), and portability changes. OpenSLP now works on OpenSLP ported to Solaris (2.6/7), Linux Alpha, Compaq Tru64. Building is now possible on non-gcc compilers.

CHANGELOG 0.7.2

  • Made significant changes that streamline main execution loop of slpd.
  • Fixed several sneeky memory leaks and memory access bugs that delt with langtags in libslp. Fixes bug #111058 - Pointer issues in slpd.
  • OpenSLP ported to Solaris (2.6/7) and Linux Alpha. Also functional on Compaq Tru64. Building is now possible on non-gcc compilers.
  • Documentation updates - codeset converted to iso-8859-1 and spell checked.
  • Modified slpd command line parameter parsing to print version and help information
  • Finalized the DA discovery routines in libslp
  • Changed libslp versioning.  (libslp versioning is being changed to be libtool compliant.  This means that OpenSLP 0.7.2 will ship with libslp.0.0.1.  The reason this looks weird is that we've been doing things the Linux way for so long that it is impossible to make the numbers match.  Everything will be synced with OpenSLP 1.0   Read libtool versioning documentation for more reasoning
